# chamchamcham
# 🎮 참참참! (Cham-Cham-Cham) - HuskyLens 얼굴 인식 게임

제작자: 이해준(당곡고 10120), 정해성(신림고 20223), 오윤서(수도여고 10915)

라즈베리파이 피코 2W와 허스키렌즈(HuskyLens)의 얼굴 인식 기능을 활용한
전통 게임 '참참참'을 재해석한 IoT 게임입니다.
스마트폰 웹 대시보드로 게임을 조작하고 결과를 실시간으로 확인할 수 있습니다.

## 📖 게임 소개

'참참참'은 상대방의 고개 방향을 맞히는 전통 놀이입니다.
이 프로젝트에서는 **피코가 랜덤으로 방향을 정해 LED를 켜고, 플레이어는 카운트다운이 끝나기 전에 고개를 왼쪽/오른쪽으로 돌려서** LED 방향과 다르면 승리하는 게임으로 구현했습니다.

### 게임 규칙

1. 스마트폰 대시보드에서 **"게임 시작"** 버튼을 누릅니다.
2. 피코가 **3, 2, 1** 카운트다운을 진행합니다. (대시보드 화면에서 확인 가능)
3. 카운트다운이 끝나면 LED 바의 **왼쪽 끝** 또는 **오른쪽 끝** LED 중 하나가 랜덤으로 켜집니다.
4. 플레이어는 카운트다운이 끝나기 전에 고개를 왼쪽이나 오른쪽으로 돌립니다.
5. 허스키렌즈가 인식한 얼굴의 좌우 위치와 LED가 켜진 방향을 비교합니다.
   - **같은 방향** → 플레이어 패배 😢
   - **다른 방향** → 플레이어 승리 🎉
6. 게임 결과는 스마트폰 대시보드에서 바로 확인할 수 있습니다.

## 🛠 사용한 하드웨어

| 부품 | 역할 | 연결 위치 |
|---|---|---|
| Raspberry Pi Pico 2W | 메인 컨트롤러 (Wi-Fi 내장) | - |
| HuskyLens | 얼굴 인식(Face Recognition) | I2C1 (SDA: GPIO6, SCL: GPIO7) |
| WS2813 LED 바 (10구) | 게임 방향 표시 및 카운트다운 표시 | GPIO16 |

> ⚠️ HuskyLens는 사용 전 본체 버튼으로 **Face Recognition 모드**로 전환해야 합니다.

## 💻 소프트웨어 구조

- **펌웨어**: MicroPython
- **비동기 처리**: `uasyncio`를 사용하여 게임 로직과 웹 서버를 동시에 실행
- **웹 대시보드**: 별도 프레임워크 없이 순수 HTML/CSS/JavaScript로 구현, 피코가 직접 서빙
- **통신 방식**: 스마트폰 브라우저가 `fetch()`로 0.5초마다 피코에 상태를 요청(polling)하는 방식

```
📁 프로젝트 구조
├── main.py           # 메인 코드 (게임 로직 + 웹서버 + LED/센서 제어)
└── wifi_config.py    # Wi-Fi 접속 정보 (개별 생성 필요, 깃허브에는 포함하지 않음)
```

## 🚀 설치 및 실행 방법

### 1. 하드웨어 연결

- HuskyLens ↔ 피코 I2C1 (SDA: GPIO6, SCL: GPIO7, VCC, GND)
- WS2813 LED 바 ↔ 피코 GPIO16 (데이터 핀), 별도 5V 전원 권장

### 2. Wi-Fi 설정 파일 생성

프로젝트 루트에 `wifi_config.py` 파일을 아래와 같이 직접 만들어주세요. (이 파일은 개인정보 보호를 위해 깃허브에 올리지 않습니다.)

```python
WIFI_SSID = "본인의 와이파이 이름"
WIFI_PASSWORD = "본인의 와이파이 비밀번호"
```

> 📌 피코는 **2.4GHz Wi-Fi**만 지원합니다. 5GHz 전용 네트워크에는 연결할 수 없습니다.

### 3. 코드 업로드

Thonny 등의 도구를 사용해 `main.py`와 `wifi_config.py`를 피코에 업로드합니다.

### 4. 실행

피코를 재부팅하면 자동으로 게임 서버가 실행되며, 시리얼 콘솔에 아래와 같은 메시지가 출력됩니다.

```
와이파이 연결 성공!
IP 주소: 192.168.0.xx
웹 대시보드 주소: http://192.168.0.xx
```

### 5. 접속

피코와 **같은 Wi-Fi에 연결된 스마트폰**의 브라우저에서 위 IP 주소로 접속하면 대시보드가 나타납니다.

## 🌐 웹 대시보드 기능

- 실시간 카운트다운 표시 (3, 2, 1)
- LED 점등 방향 안내
- 게임 결과(승/패/무승부) 표시
- 인식된 얼굴의 X좌표 값 표시 (디버깅 겸 참고용)
- "게임 시작" 버튼으로 언제든 새 게임 시작 가능

## ⚙️ 주요 설정값 (main.py 상단)

| 변수명 | 설명 | 기본값 |
|---|---|---|
| `LED_PIN` | LED 바 데이터 핀 번호 | 16 |
| `NUM_LEDS` | LED 개수 | 10 |
| `BRIGHTNESS_MAX` | LED 최대 밝기 (0~255) | 60 |
| `HUSKYLENS_FRAME_WIDTH` | 허스키렌즈 화면 가로 해상도 | 320 |
| `DEAD_ZONE` | 얼굴 좌우 판정 시 중앙 여유 구간(픽셀) | 30 |
| `INVERT_LEFT_RIGHT` | 좌우 판정이 반대로 인식될 경우 `True`로 설정 | False |

## 🔧 트러블슈팅

- **스마트폰에서 대시보드 접속이 안 될 때**
  - 피코와 스마트폰이 정확히 같은 Wi-Fi(같은 대역, 2.4GHz)에 연결되어 있는지 확인하세요.
  - 일부 공공/학교 Wi-Fi는 기기 간 통신(AP Isolation)을 차단할 수 있습니다. 이 경우 스마트폰 핫스팟으로 테스트해보세요.

- **`OSError: [Errno 98] EADDRINUSE` 오류가 발생할 때**
  - 이전 실행에서 포트가 완전히 정리되지 않은 경우 발생합니다. 피코를 재부팅한 뒤 다시 실행하세요.

- **고개 방향 인식이 부정확할 때**
  - 시리얼 콘솔에 출력되는 얼굴 X좌표 디버그 값을 확인하고, `DEAD_ZONE` 값을 조정하세요.
  - 좌우가 반대로 인식된다면 `INVERT_LEFT_RIGHT`를 `True`로 바꿔보세요.
